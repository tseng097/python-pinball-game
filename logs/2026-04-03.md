# 2026-04-03

## Observed issues
- Flippers apply continuous motor torque even when released, which can keep them pushing into the rotary limit and introduce jitter/energy injection.
- Released flippers can take a while to settle at the rest angle because there’s no explicit return spring.
- Headless tests currently fail in this environment because `pymunk` is not installed.

## Changes
- Added a damped rotary spring to each flipper to pull it toward the rest angle when released.
- Reduced motor drive when released by setting motor rate to 0 and lowering max force.
- Added a new headless test asserting a released flipper returns to rest with low angular velocity.

## Parameter diffs
- `settings.py`:
  - Added `FLIPPER_SPRING_STIFFNESS = 120000`
  - Added `FLIPPER_SPRING_DAMPING = 8000`
- `entities.py`:
  - `Flipper.set_active(False)` now sets motor rate to `0.0` and max force to `FLIPPER_MAX_FORCE * 0.2`.
  - Added `pymunk.DampedRotarySpring` for flippers.

## Tests
- `python3 -m pytest -q` → failed (pytest not installed)
- `python3 -m unittest -v` → failed (no tests discovered)
- `python3 -m unittest discover -s tests -v` → failed (missing `pymunk` module)

## Next steps
- Install test dependencies (`pymunk`, optionally `pytest`) in CI/dev env so headless tests run.
- Tune spring stiffness/damping if flippers feel too soft or too stiff in gameplay.
