# 2026-04-01

## Observed issues
- Physics stepping uses the raw frame dt with fixed substeps, so a frame hitch can create oversized steps that let the ball tunnel or jitter through walls/bumper gaps.
- Tests cover stability at nominal dt, but there was no automated check asserting that large dt values are safely clamped.

## Changes
- Clamp physics step dt to a maximum safe value before substepping.
- Added a headless test to ensure large dt values are clamped and do not move the ball excessively in a single step.

## Parameter diffs
- `PHYSICS_MAX_DT`: new constant set to `1/30.0`.

## Test results
- `python3 -m unittest discover -s tests -p "test_*.py" -q` → **FAILED** (ImportError: `pymunk` not installed in environment).

## Next steps
- Install `pymunk` in CI/dev environment so physics tests can run.
- Consider increasing substeps or adaptive substepping if tunneling persists at very high speeds.
