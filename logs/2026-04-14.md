# 2026-04-14 Pinball Physics Daily Log

## Observed issues
- Flipper motor stays fully powered even when the flipper is already at its max angle. With motor rate set to 0 but max_force still high, the motor can fight constraints/collisions and create jitter or extra energy loss.
- Test runner unavailable in environment (pytest missing), so automated checks couldn't be executed today.

## Changes
- Disengaged the flipper motor when the flipper reaches its max angle so the rotary limit + spring do the holding, reducing motor fighting at the stop.
- Added an automated assertion ensuring the motor max_force is zero when the flipper is at its limit.

### Parameter/behavior diffs
- `Flipper.set_active(True)` now sets `motor.max_force = 0.0` when `angle >= max_angle - FLIPPER_LIMIT_BUFFER`.

## Next steps
